分析一下二叉搜索树的高度

因为东大的考试题上竟然出现了这样变态证明题，所以着重看了一下。

randomly built binary search tree's height。这个怎么理解呢，就是序列是一个uniform random distribution的序列，那么这个通过这个序列，用insert的方法建成了binary search tree的高度的期望是多少。

先说一下书上的思路，猜高度为O(lgn)，通过证明高度的指数次幂函数为多项式，取log就为O(lgn)。

因为root是随便取的，而均一分布的序列每个元素都有同样的机会取到，在某个特定序列下，如果X(n)是n个节点的树的高度，i这个节点是根，那么：X(n)=max(X(i-1),X(n-i))+1. 把随机变量i加进去 E[Xn] = SUM( i=1..n , E[Zn,i])，Zn,i表示n个节点恰好选择i作为根的事件，很明显这个事件的概率为1/n，而且相互独立。E[Xn] = 1/n*SUM(i=1..n,E[1+MAX( Xi-1,Xn-i)] )。为了求这个，换个思维，设 exponential height: Yn = 2^Xn。Yn=2*MAX(Xn-i,Xi-1)。好了我们的公式变为 E[Yn] = 1/n*SUM( 2*E[ MAX(Yi-1,Yn-i) ]); E 可以进去。现在就剩MAX(E[Yi-1],E[Yn-i])不好处理了，用缩放把MAX变为+。MAX(E[Yi-1],E[Yn-i]) <= E[Yi-1] + E[Yn-i]。注意角标 i-1 和n-i。可以认为做了2次E[Yi-1]的运算。整理一下

E[Yn] = 4/n* SUM(i=0..n-1, 2*E[Yi]);

好了，现在才到了难点，这个东西怎么解。书上做了一个超级无敌归纳法假设：E[Yn] <= 1/4* C(n+3,3)。C是组合数

我勒个去，你很想问这个是怎么来的，1/4，3这些数又是怎么来的。首先我们想要证E[Yn]是个多项式。C不就是多项式么，别管系数多少，取个log全没有了。组合里面有个 C(n,r) = C(n-1,r)+C(n-1,r-1)。这个公式，通过这个公式，把C(n-r,r)一直向后展开，可以把SUM消掉。然后我们可以写出一个E[Yi] = a*C(n+i,i)的假设。带入E[Yn]公式，E[Yn] = 4/n* SUM(i=0..n-1, 2*E[Yi])。发现分母恰好多一个d+1，跟4/n的4碰上了，d可以得到3。你可能问a还没算呢，那a不是随便取么。归纳法还有个初始值的条件，把E[Y1] = 1带进去，a算出来得4。

然后现在方便多了，E[Yn] = O(n^3)。还没有完。Yn = 2^Xn，和期望还没有扯上关系。需要一个延森不等式Jensen's inequality，E[f(x)] >= f(E[x])，当f(x)是凸函数的时候。

E(Yn)>=2^E(Xn)

E(Yn)的公式带入，2^E(Xn)<=n^3的公式，取log，得到O(lgn)。

这个证明是很数学的，数学就是猜啊。

不知大家有没有更好的证明方法。

你是怎么写随机算法的

昨天才知道我一直写的随机算法是错的……

你是怎么生成一个随机序列的，比如n的全排列中以1/n!概率选取一个串。

看下面两段程序：

RANDOMIZE-IN-PLACE 

1 n = A:length

2   for i = 1 to n

3   swap a[i] , a[rand(i,n)]

其中rand(i,n) 表示 随机产生一个[1..n]之间inclusive 的数。

PERMUTE-WITH-ALL

1 n = A:length

2   for i = 1 to n

3   swap a[i] , a[rand(1,n)]

看起来好像没多大区别，一个从i..n之间挑一个，一个从1..n之间挑一个。第二个貌似比第一个还要随机一些。

先对第一个分析，假设第 i 次 使1..i 位置上的元素 成为 [X1..Xi] 的概率为（n-i）!/n! 这个概率是怎么来的呢，从n个数里选i个数带排列的概率 1/A(n,i)。 

当算i+1的时候，i+1这个位置只能从n-i-1+1（包含自己）个位置随机一个了，假定选到Xi+1，那么随机它的概率就是1/n-i-1。（n-i）!/n! * 1/n-i =(n-i-1)!/n!。 所以加上Xi+1 这个元素，公式依然成立。

最后概率为i=n P= 1/n!

这个算法最大的特点是保证了，每到i位置时，获得到数列的概率就是从n个数中选择i个数的概率。

再来看第二个，这个貌似更随机的会怎样。

因为rand(1..n) 的结果 每次有n种可能，也就是1..n种选择，运行n次。也就是n^n的sample space。每一个元素 in sample space 都对应唯一的permutation。 哪有多少种permutation呢？ n! 种。所以要想让这n!种有同样的概率，（因为sample space的每个元素的概率都是一样的），sample space里面元素 到每一种permutation的元素的对应应该一样多，也就是 x*n! = n^n。 

但是x这样的整数 不一定存在，比如n=3。所以，这个方法不能产生均一分布的序列 ( uniform random permutation ) 。

昨天编程试了一下，果然不是均一的，至于怎么分布的没有弄明白。n=4的情况下有的概率可达14/4^4，有的只有8/4/^4，差别还是比较大的。

呵呵，有意思吧，我以前一直总写这个错的。另外还有一种方法，每次随机2个数i,j，随机n次，每次交换a[x],a[y]。这个方法和上面那个是一样的，也不能产生 uniform random permutation。（这一部分是我自己想的，有可能有误）

就从分析 hire assistant problem 开始吧 ( 概率分析，随机算法 )

问：现在要雇一个assistant，有n个人来面试，每次面试到能力高的就把前面雇佣的assistant解雇，然后雇佣这个能力高的人。n个人的能力分别1..n。求（平均）解雇/雇佣的次数。

给出程序看得更舒服点儿：

1 best D 0 / / candidate 0 is a least-qualiﬁed dummy candidate

2 for i D 1 to n

3    interview candidate i

4      if candidate i is better than candidate best

5        best D i

6        hire candidate i  

求第六行 执行多少次。

这种cost和输入的问题，一般只讨论average running time。求第六行执行多少次 = 求期望。

Exception = sum ( xi * Pr{x = xi} )

用这个公式套这道题，xi 表示恰好雇佣 xi 次人。那么恰好雇佣 xi 次乘以相应的概率，就是平均雇佣次数了。

但是，为了方便转化 期望和概率之间的关系，发明了一个叫indicator random variable 的东西，值域只有1,0 发生就是1，不发生就是0。

E[x] = sum(E[Xi]); //自己看书吧 page 120

用这个方法，对于这个问题 Xi 就表示 站在i位置上的人被雇佣 的事件。P(Xi)就好求了，站在i位置上的人 要想被雇佣，i 上这个人的能力值一定得最大，所以1..i上最大的那个数必须在i位置上，其他无所谓，所以P(Xi) = (i-1)!/i! = 1/i。 期望就是 1..n 位置上的人 被雇佣的期望的和。

E[x] = sum (1/i)

E[x] = ln(n); // 相当于对 1/x 的积分， 不会的去看 微积分。
